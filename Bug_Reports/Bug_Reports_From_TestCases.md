Поле "Имя" должно содержать минимум 2 символа. Имя из одного символя не должно приниматься формой.

## BUG-006 — API валидация не работает (пустое имя) {#bug-006}

| Поле | Значение |
|------|----------|
| **Номер** | BUG-006 |
| **Заголовок** | API создаёт заявку с пустым именем клиента |
| **Severity** | Critical |
| **Priority** | Критический |
| **Статус** | Open |
| **Компонент** | Backend / Bookings API |
| **Тест-кейс** | API-004 |

---

### Предусловия:
1. Backend запущен на `http://localhost:8080`
2. База данных подключена

---

### Шаги воспроизведения:

| № | Действие |
|---|----------|
| 1 | Открыть Postman или аналогичный инструмент |
| 2 | Отправить POST запрос на `/api/bookings/` |
| 3 | В теле запроса указать пустое поле customer_name: `""` |
| 4 | Заполнить остальные поля корректно |
| 5 | Отправить запрос |

**Пример запроса:**
```json
{
  "customer_name": "",
  "customer_phone": "+7 (999) 123-45-67",
  "car_brand": "Toyota",
  "car_model": "Camry",
  "service_id": "svc_1",
  "comment": "Test"
}
```

---

### Ожидаемый результат:
- Status code: 400 Bad Request
- Тело ответа содержит ошибку валидации
- Заявка НЕ создаётся в базе данных

### Фактический результат:
- Status code: 201 Created
- Заявка создаётся в базе данных с пустым именем
- Ошибок валидации не появляется

---

### Требование:
API должен отклонять заявки с пустым именем клиента. Валидация на уровне backend обязательна.

---

### Воспроизводимость:
- [x] Всегда (100%)
- [ ] Часто (>50%)
- [ ] Иногда (<50%)
- [ ] Не воспроизводится

---

### Примечания:
- **Критическая уязвимость** — обход валидации frontend
- Требуется немедленное исправление
